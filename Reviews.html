Code Reviews

My name is Earlz, and I do code reviews of altcoins. My primary focus is on security, not the feature of the month that each coin implements. For instance, I don't care about IRC or exchanges in wallet. Basically I compare the code of each altcoin in relation to a "known-safe" coin such as Bitcoin and look at the differences between them, and examine the network and launch conditions.

My complete, but not always up to date, list of code reviews is at github

Some of the things I've spotted in the past:

  • Numerous undisclosed premines (mining started before public launch)
  • Purposefully disabled difficulty adjustment to enable insta-mining at launch
  • An exploit that enabled the developer to mine a block at anytime at will and skip PoW checking
  • An exploit that enabled the developer to mine a block with 1 billion coins (When the reward was suppose to be 100 coins)
  • Zero reward PoW, which made it trivial to 51% attack

Along with these critical issues, I also spot minor issues that leave a coin in a state that isn't as secure as it could be. The way my reviews work is I give a list of Problems with a severity rating. Mostly anything that's "MODERATE" severity and below is not much to worry about. I do point out these minor issues while I'm looking though to explain that the issue exists and that it should be fixed if possible.

Release

I must have all funds up front before I can start the review. Once the review is completed, if there are any critical issues then I must post it. Depending on the exact issue, I may first notify exchanges before publishing on the announcement page. This is to ensure that in the case of a malicious developer, the coin can be delisted before they have a chance to cash in. If there are no critical issues, I will post it immediately to the announcement. In some cases I may give the developer heads up a little bit before posting, to ensure that they understand the problems listed.
